## Infrastructure learnings (worth preserving)

**Bot-launch pattern that actually works from an interactive
session.** The `Start-Process cmd /c scripts\run_session.bat`
pattern via PowerShell inherits the parent shell's env — which
means a freshly-setx'd user env var is NOT visible to the spawned
cmd. Two gotchas hit today; the durable fix is:

```powershell
$key = # read OPENAI_API_KEY from MiroShark .env
$psi = New-Object System.Diagnostics.ProcessStartInfo
$psi.FileName = "cmd.exe"
$psi.Arguments = "/c scripts\run_session.bat <budget> openrouter"
$psi.WorkingDirectory = "C:\Users\rweis\OneDrive\Documents\GeneralStaff"
$psi.UseShellExecute = $false
# CRITICAL: copy Machine + User + Process scope env
foreach ($e in [Environment]::GetEnvironmentVariables("Machine").GetEnumerator()) {
  $psi.EnvironmentVariables[$e.Key] = $e.Value
}
foreach ($e in [Environment]::GetEnvironmentVariables("User").GetEnumerator()) {
  $psi.EnvironmentVariables[$e.Key] = $e.Value
}
foreach ($e in [Environment]::GetEnvironmentVariables("Process").GetEnumerator()) {
  $psi.EnvironmentVariables[$e.Key] = $e.Value
}
$psi.EnvironmentVariables["OPENROUTER_API_KEY"] = $key
[System.Diagnostics.Process]::Start($psi)
```

Without Machine-scope copy, system PATH is missing and PowerShell-
based tests fail in the bot's worktree. Without the explicit API-
key injection, the reviewer cycles fail with OPENROUTER_API_KEY
missing.

**`UseShellExecute = false` hides the cmd window.** Ray likes the
visible window as ambient confirmation the bot is actually running.
Tradeoff accepted for this session; worth revisiting — maybe write
a helper .bat wrapper instead so the window stays visible without
losing env control.

**Preview worktree pattern for bot/work review:**
`git worktree add --detach ../gamr-preview <sha>` so the preview
holds a commit, not the branch name. Otherwise the bot's own
`.bot-worktree` creation fails with git exit 128 because the branch
is already checked out.
