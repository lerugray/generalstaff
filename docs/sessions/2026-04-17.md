## Key lessons captured for future sessions

- **Task spec vs hands-off list can conflict** (gs-119 stop
  --force, gs-125 design doc, gs-146 stop --status — all
  natural implementations hit hands-off files). Task drafting
  should anticipate implementation surfaces and explicitly
  guide alternatives. gs-165's task description includes
  "Do NOT modify src/safety.ts" with a pointer to the correct
  inline pattern.
- **Reviewer reliability has tiers.** Ollama = false-positive
  risk. OpenRouter = JSON-parse-failure risk. Neither is a
  correctness issue per se, but both waste cycle budget.
  Calibration harness (gs-156) provides a gate test before
  future provider swaps.
- **gs-132's rollback makes hands-off violations cheap.** Before
  gs-132, a hands-off violation was an expensive loss (bad code
  landed on master). After gs-132, it's a cheap loss (rolled
  back, cycle budget consumed, master clean). That shifts what
  kinds of tasks are safe to queue — borderline tasks that
  might hit hands-off can now be tried without risk.
- **End-of-session Ingest is the formalization of something we
  were already half-doing.** Karpathy's wiki-gist pattern is the
  schema (raw sources + LLM-wiki + schema document =
  CLAUDE.md). The convention is now explicit; this file is
  itself an Ingest artifact.
