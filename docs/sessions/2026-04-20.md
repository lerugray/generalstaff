## The journey — what we learned

### Three fixes that mattered

1. **`--edit-format udiff`.** Qwen3 Coder Plus via OpenRouter
   emits code fences with **four backticks** (````typescript)
   instead of three. Aider's `whole` and `diff` formats both
   parse backtick-bounded blocks — neither recognized the
   four-backtick shape, so aider silently dropped every edit.
   First two smoke runs: 100% empty_diff. Switching to `udiff`
   (standard unified-diff format, no backticks) fixed parsing
   immediately — gamr-021 went from empty_diff to verified in
   30 seconds.
2. **`PYTHONIOENCODING=utf-8`.** Aider's `rich` library crashes
   on Windows with cp1252 the first time the model emits a
   non-ASCII character (≥, ✓, █, etc.). `--no-pretty
   --no-fancy-input` disable the TUI but *not* the
   token-usage and progress bars that still emit unicode. 3 of
   10 benchmark tasks (gamr-028/029/030) failed for this reason.
   Fix: set `PYTHONIOENCODING=utf-8` and `PYTHONUTF8=1` in the
   generated bash. Unconditional because safe on every OS.
3. **Detach HEAD in the benchmark temp clone after injecting
   the task.** First version of the harness checked out
   `bot/work` in the temp clone, which then blocked aider's
   `git worktree add bot/work` with "already used by worktree."
   Fix: `git checkout --detach HEAD` after the injection commit
   so `bot/work` is a ref, not a checkout. Lives in
   `injectSingleTaskQueue`.

- **Ray delegates structural code decisions to Claude in

  already governs bot cycles: execution choices belong to the
