### Dogfooding decision (commit 789699d)

Key decision: instead of using catalogdna as the first test
project, Ray proposed dogfooding — making GeneralStaff its own
first managed project. Rationale:

1. Avoids interfering with active catalogdna work
2. Exercises the greenfield path (new project, no legacy)
3. Self-referential test: if the dispatcher can improve its own
   codebase, that's a strong signal
4. Lower risk: no production consequences if cycles fail

This turned out to be the right call. The bot immediately had
real work to do (test coverage, code quality improvements) and
the feedback loop was tight — bugs in the dispatcher were
surfaced by running the dispatcher.

### Worktree isolation

Bot works in `.bot-worktree` on the `bot/work` branch. Ray
stays on `master`. No branch-switching conflicts. Merges from
bot/work into master happen between cycles. This is the same
pattern catalogdna uses, validated over 22+ bot runs there.

### Bug #1: branch-awareness (commit 26bbf2b)

First live cycle surfaced a real bug: `cycle.ts` captured SHAs
from HEAD (which was master, the interactive branch) instead of
from the bot/work branch where the engineer actually committed.
Result: the diff passed to the reviewer was always empty. The
reviewer correctly identified this as "nothing to verify" and
returned a weak/failed verdict.

This is a textbook "initial negative" — the system surfaced its
own bug on the very first run. Fixed by making cycle.ts
branch-aware: it now reads SHAs from bot/work explicitly.

### Bug #2: auto-commit of state artifacts (commits 0467d53, b089730)

Between cycles, the dispatcher auto-commits state artifacts
(fleet_state.json, cycle logs). Two issues:

1. First fix: auto-commit wasn't happening at all between cycles
2. Second fix: `git add` failed on gitignored paths. Switched
   to `--ignore-errors` flag for state artifact commits.

Both were plumbing issues, not architectural problems. Fixed
within the same session.

## Key decisions

1. **Dogfood over catalogdna** — GeneralStaff builds itself as
   its first test project. Avoids cross-project risk, exercises
   greenfield path, tighter feedback loop.
2. **Worktree isolation** — bot/work branch in .bot-worktree,
   master stays clean for interactive work. Proven pattern from
   catalogdna.
3. **Branch-aware cycle orchestration** — after bug #1, cycle.ts
   explicitly targets bot/work for SHAs and diffs instead of
   reading from HEAD.
4. **State artifact commit strategy** — auto-commit between
   cycles with --ignore-errors for gitignored paths.
5. **14-module architecture** — clean separation of concerns:
   CLI, types, state, safety, work detection, audit, engineer,
   reviewer (+ prompt template), dispatcher, session,
   verification, cycle, projects.
