Cycle phases

Every cycle runs engineer → verify → review. Engineer is the Claude Code session doing the work; verify is bun test && tsc --noEmit; review is the verdict model reading the diff against the claimed task. Any phase can fail and roll the cycle back.

Verdicts

verified means the diff matches the claimed task with no hands-off violations. failed means the reviewer rejected the diff or verification didn't pass. A rolled-back cycle had its changes reverted on the branch.

Stop reasons

Sessions end on max-cycles, budget-elapsed, no-work, or the STOP file. The summary band above shows how tonight's sessions closed. Read-only here — start and stop sessions from the CLI.
