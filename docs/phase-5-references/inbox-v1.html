What this channel is

An append-only JSONL at state/_fleet/messages.jsonl. Sessions post structured breadcrumbs — status, handoffs, blockers, decisions — that later sessions can read at startup. Not a chat room; closer to a ship's log.

Message kinds

handoff passes queue-state between sessions. blocker flags something a later session should address. fyi is informational. decision is a human call worth preserving. Kinds are a convention, not enforced by schema.

Retention

Reads from a session-start cutoff so restarts don't re-log ancient history (gs-219). The file itself is append-only — old messages stay on disk as a durable audit trail alongside PROGRESS.jsonl. Trim by hand if it ever grows noisy.
