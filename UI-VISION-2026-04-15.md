## Theme: Prussian Kriegspiel / command room

The local UI should have an iconic visual identity tied to the
GeneralStaff / Hammerstein framing. The reference is the Prussian
**Kriegspiel** — the 19th-century war-game training system used
at the Kriegsakademie by the Prussian general staff. Picture a
campaign room with map tables, terrain blocks, dispatches arriving
from the field, a duty officer marking pieces forward.

This metaphor is right because it matches the project's actual
operating model:

| Kriegspiel concept | GeneralStaff equivalent |
|---|---|
| Front on the campaign map | Registered project in `projects.yaml` |
| Dispatch from the field | Cycle report (PROGRESS.jsonl entry) |
| After-action report | Reviewer verdict (REVIEW.md) |
| Forbidden ground / no-fire zone | Per-project hands-off list |
| Staff officer checking dispatches against ground truth | The verification gate (Hard Rule #6) |
| Daily morning briefing | Morning digest |
| Stand-down order | `STOP` kill-switch file |
| Command (the user) | Sets direction, reads briefings, makes strategic calls |
| The general staff (the bots) | Handles execution where industriousness compounds |

## Why this works as a brand

1. **It matches the philosophy.** Hammerstein's typology is itself
   Prussian general-staff doctrine. The visual identity comes from
   the same intellectual tradition as the underlying philosophy.
   Not borrowed; inherited.
2. **It is not what other agent platforms look like.** Polsia,
   Lovable, Cursor, Devin all share a flat-modern-SaaS aesthetic.
   A kriegspiel-themed open-source competitor stands out
   immediately in screenshots and demos.
3. **It encodes the rules visually.** The campaign-room metaphor
   makes "this is the staff handling logistics; you remain in
   command" obvious without any reading.
4. **It is fun to build.** Visual design is the reward for the
   engineering rigor of the rest of the product. Ray cares about
   taste — this is where taste shows up.

## Aesthetic notes (when the time comes)

- Muted, lithograph-style — paper, ink, leather, brass fittings.
  Not glass-and-neon SaaS.
- Map view (fleet overview) with project pins on a stylized
  campaign-map background
- Dispatch view (per-project drilldown) styled as a written report
  in a typewriter / fixed-width font
- Audit log view styled as a war diary — chronological, immutable,
  permanent
- Palette inspired by Prussian uniforms: dark blue, black, brass,
  cream paper. *Not* a literal military aesthetic.
- Open-source fonts in the period tradition: Bodoni / Didot /
  Caslon family for serif display, monospace for data

## What this is NOT

- Not military cosplay
- Not an aesthetic statement about war or violence — the reference
  is the *staff training tradition*, not the wars themselves
- Not a literal recreation of any historical interface
- Not a constraint on Phase 4-5 functionality — get the mechanics
  right first; theme later

The reference is the *intellectual tradition* (operational
planning, professional general staff, war-game training), not the
iconography. **No eagles, no iron crosses, no overt symbols.**
Those would read as edgy or cosplay and miss the entire point.

## When this happens

After Phase 5 (the control surface ships and works), call it
Phase 5.5. Slot in when the mechanics are stable. Don't theme a
moving target. Until then this file is just intent.

## Reference points (no commitment)

- The Avalon Hill / SPI / GMT campaign games' map aesthetic
- Old illustrated military atlases from the 1880s-1900s
- The actual Kriegsakademie dispatch-room photography
- Prestige-film opening-title sequences that use lithograph maps
- Edward Tufte's information design — particularly Minard's 1869
  Napoleon-in-Russia chart, which is itself a Prussian-adjacent
  staff-officer artifact and the canonical example of dense data
  presented honestly
